Two ways to use Pax

Pick the tier that matches how you want to consume verified knowledge. Tier 1 needs no tooling. Tier 2 hands paxes to an agent via the Praxis MCP server.

{{- /* ── Tier tab nav ───────────────────────────────────────────────── */ -}}
{{- /* ── Tier 1: Paste-to-Claude ─────────────────────────────────────── */ -}}
{{ partial "section-head.html" (dict "caption" "tier.01 · zero install" "title" "Paste-to-Claude") }}

Copy a pax bundle into any chat-based LLM. Works with Claude, ChatGPT, Gemini, Llama. No tooling, no account, no install.

  1. 1
    Pick a pax from /pax/ and click Download. You’ll get a .pax.tar.gz archive (1–100 KB).
  2. 2
    Extract and open llm-context.md. This file contains the structured constructs, findings, and citations.
  3. 3
    Paste the contents into your AI chat as the first message. Then ask questions — the AI will cite findings by ID.
// what you can ask
  • What are the top 3 findings on construct risk_exposure_index?
  • Cite all findings with p < 0.05 and effect size above 0.3.
  • How do constructs cascading_failure_size and TTR relate?
  • Summarize the evidence for proposition P001 — quote it verbatim.
markdown · llm-context.md 
SYSTEM: You are reasoning over the attached pax.
Constraints:
  - Cite findings by ID (F001, F042, ...)
  - Quote propositions verbatim when used
  - If the pack does not cover a question, say so

<pax name="global-supply-chain-risk" version="1.0.9">
constructs:
  - id: risk_exposure_index
    kind: quantifiable
    units: ratio
    def: TTR/TTS ratio for each node in the supply graph
  - id: cascading_failure_size
    kind: quantifiable
    units: fraction_of_network
    def: share of downstream nodes disrupted after removal

findings:
  - id: F001
    text: |
      Supply chain disruptions cause average abnormal
      stock returns of approximately -40% over a
      three-year window post-event.
    beta: -0.40
    p:    "<0.001"
    n:    827
    source: hendricks_singhal_2005
  - id: F003
    text: |
      High betweenness-centrality nodes (top 5%) account
      for 64% of cascading failure propagation paths.
    ...
</pax>

USER: Identify the top-3 vulnerabilities in our
network given:
  - 14 tier-1 suppliers, 6 chokepoint-exposed
  - mean inventory: 18 days
  - revenue at risk: $2.1B/yr
{{- /* ── Tier 2 (was Tier 3): MCP server via Praxis ──────────────────── */ -}}
{{ partial "section-head.html" (dict "caption" "tier.02 · agentic" "title" "MCP server (Praxis)") }}

Run Pax inside any MCP-compatible agent (Claude Code, Continue, Zed, Cursor) via Praxis, the reference MCP server for the pax format. Tools like praxis_search_pax, praxis_install_pax, praxis_query_findings become first-class.

{{ partial "section-head.html" (dict "caption" "install" "title" "Run Praxis locally") }}

Praxis is an open-source Python MCP server. Clone, install, and point your agent at it. Setup instructions live in the Praxis README.

{{ partial "section-head.html" (dict "caption" "exposed tools" "title" "Tool reference") }}
tool signature description
praxis_search_pax (query: str) search the registry by keyword
praxis_install_pax (pax: str) download and install a pax locally
praxis_query_findings (construct: str, pax?: str) retrieve findings for a construct
praxis_query_constructs (pax?: str) list and search constructs across paxes
praxis_align_constructs (pax_a: str, pax_b: str) find bridge constructs between two paxes
praxis_run_playbook (pax: str, playbook: str) execute an analytical playbook

──────── results ────────

cited: F001, F003, F005, P001.
